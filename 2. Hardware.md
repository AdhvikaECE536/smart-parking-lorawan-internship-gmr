### Overview

This board is buried under an airport parking bay. It detects vehicles via magnetic disturbance and reports occupancy (1 = occupied, 0 = vacant) over LoRaWAN to a ChirpStack dashboard.

The board, labelled STATIONS-V1.1, is a small, battery-powered PCB designed to live inside a
waterproof enclosure flush with the road surface. 
On it are passive components like resistors and capacitors and:

1. STM32WLE5CCU6 is a single-chip System-on-Chip cobining 32-bit Cortex-M4 CPU and a sub-GHz LoRa radio transceiver.
    - The STATIONS-V1.1 board uses the STM32WLE5CCU6 in its bare QFN-48 form factor, not a pre-integrated module. While module implementations (such as the MS23SF14) consolidate all supply pins into a single VDD pin, this design exposes the chip's internal power domains separately to allow precise decoupling and RF noise isolation. The 1.8 V LDO output feeds VDD1/VDD2 directly, while VDDA is filtered via ferrite bead FB1, and VDDRF/VDPA have dedicated capacitor banks for radio stability.
    - **WHY**: It runs the firmware, reads the sensor over I2C, decides if a car is present, builds the encrypted LoRaWAN packet, and sends it into the antenna.

2. BMM350 is a 3-axis magnetometer from Bosch
    - **WHY**: It constantly feels the Earth’s magnetic field along X, Y, and Z. When a car parks above it, the steel mass distorts the field.
    - The BMM350 can be configured to pull an interrupt pin low when the magnetic field crosses a threshold. This lets the STM32 sleep at micro-amps and only wake when a car actually arrives, instead of polling the sensor every second and draining the battery

3. TPS7A0218PDBVR is a tiny Low-Dropout (LDO) voltage regulator.
    - **WHY**: When the battery delivers roughly 3.0–3.7 V. The STM32 and BMM350 both need exactly 1.8 V. The LDO acts as a quiet translator: it drops the battery voltage to a clean, stable 1.8 V without introducing switching noise that could disturb the radio or magnetometer. 
  
4. BGS12WN6E6327XTSA1 is an SPDT (Single-Pole Double-Throw) RF switch
    - It is between the STM32’s radio output and the antenna matching network.
    - **WHY**: Routes the radio signal to either the on-board chip antenna (RF1, normal use) or a future external test port (RF2, currently unpopulated/DNP). This lets engineers test or certify the radio without redesigning the PCB.

5. M620720 is a passive ceramic chip antenna
    - **WHY**: It turns the electrical signal from the radio into an electromagnetic wave that flies through the air to the gateway. It is flat and rugged—unlike a wire whip that could snap under a tyre or cleaning machine. The empty green rectangle around it is intentional: metal traces nearby would blind the antenna in some directions.

6. CSO6654-32M is a 32 MHz precision quartz crystal resonator.
    - **WHY**: LoRa modulation requires extreme frequency accuracy—parts per million. The 32 MHz crystal provides the stable heartbeat that the radio uses to generate its chirp. Without this crystal, the radio would drift off-channel and the gateway would not hear it.

7. 32.768 kHz (RTC) Crystal is a smaller, slower quartz crystal
    - **WHY**: While the CPU sleeps deeply, this crystal keeps ticking at micro-amps. It is the Real-Time Clock (RTC) that wakes the STM32 every 4–6 hours for a heartbeat. The value 32.768 kHz is chosen because it is exactly 2^15 Hz—easy to divide down to 1-second intervals using binary counters. It is the “alarm clock” that works while everything else is off.

8. S2B-XH-A is a 2-pin JST-style header battery connector
    - **WHY**: The board is battery-powered and must be field-serviceable. A connector lets maintenance swap the battery without soldering. The “XH” footprint is a standard JST size, so replacement cells are easy to source.

9. 6-pin SWD Programming Header
    - **WHY**: This is how the engineering team flashes firmware onto the STM32 and debugs it. SWD uses only 2 data pins (SWDIO and SWCLK), leaving more GPIOs free for the application. It is the “USB port” of the embedded world.

10. Ferrite beads FB1, FB2, FB3
    - **WHY**: These are tiny inductors that block high-frequency noise. FB1 sits between the main 1.8 V rail and the analog VDDA pin, acting like a “noise gate” that keeps digital switching garbage away from the sensitive radio circuits.

11.  I2C Pull-up Resistors
    - **WHY**: I2C is open-drain; devices can only pull lines low. These resistors provide the “high” state so the bus does not float.

### Why Magnetometer?

Comparing to Ultrasonic and IR sensors, we use magnetometer with a simple, clear distinction


**Ultrasonic**: It emits high-frequency sound and waits for the echo to return and calculates the distance using this. 
- When distance drops suddenly, the sensor decides an object is present. But small acoustic sounds like from rain puddles, plastic bags, or birds can create false triggers.
- Also, in a parking bay, the pulse can hit the curb, a wheel, and the undercarriage simultaneously, producing multiple overlapping echoes (multi-path interference). The firmware sees a jumble of return times and either reports a false positive or drops the reading entirely.
- They need a line-of-sight to the vehicle undercarriage. If mounted flush with the road, the beam points upward. A low-slung sports car vs. a high SUV changes the reflection profile unpredictably. The sensor must be calibrated per vehicle type, which is impossible in a public lot.

**IR**: Sunlight interference, dirt on the lens, limited range.
- Magnetometer: No moving parts, no optics, waterproof, and it only triggers on metal mass—
exactly what a car is.
